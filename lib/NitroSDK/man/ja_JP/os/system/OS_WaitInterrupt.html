OS_WaitInterrupt

構文

#include <nitro/os.h>

void OS_WaitInterrupt( BOOL clear, OSIrqMask irqFlags );

引数

clear チェックフラグをクリアするかどうか
irqFlags 発生を待つ割り込みのマスク値。論理和で複数を指定可能

返り値

なし。

説明

指定のIRQ割り込み要因の発生を待ちます。
irqFlags には複数の割り込み要因をマスク値の論理和で指定することが出来ます。この中のいずれかの割り込みが発生したときに関数から戻ります。

割り込みが発生したかどうかは、IRQ割り込みが発生した後の割り込みチェックフラグがセットされているかどうかによります。このフラグは割り込みハンドラの中でユーザが明示的に OS_SetIrqCheckFlag() を呼んでセットする必要があります。

clear を TRUE にすると、割り込み発生待ちの前に割り込みチェックフラグのうち、irqFlags で指定された割り込みのフラグをリセットします。リセットしない場合であらかじめ該当の割り込みのチェックフラグがセットされている場合、すぐに関数から戻ることになります。

OS_WaitInterrupt() と OS_WaitIrq() との違いは、OS_WaitInterrupt() は割り込み要因を待つ間、OS_Halt() で停止しますが、OS_WaitIrq() は割り込み要因を待つ間は他のスレッドに処理を移し、割り込みが発生した後に動作を再開するというものです。スレッドを使用しない場合は、両者は同じ動作をする関数となります。
