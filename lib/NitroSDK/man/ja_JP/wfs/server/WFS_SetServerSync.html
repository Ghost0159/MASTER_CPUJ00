WFS_SetServerSync

構文

#include <nitro/wfs.h>

void WFS_SetServerSync(WFSServerContext *context, int bitmap);

引数

context WFSServerContext構造体のポインタ
bitmap 応答を同期させる子機群のAIDビットマップ
0を指定すると応答同期は行われません。(この動作がデフォルトです)
また、最下位ビット 1 は常に無視されます。

返り値

なし

説明

子機からの読み込みリクエストに親機が応答する際の同期グループを設定します。
WFSライブラリはこの関数で指定された子機群が親機にとって同じタイミングで同じ内容の読み込みリクエストを発行してくるものと仮定し、 該当する全ての子機から読み込みリクエストが発行されるまで応答を保留するようになります。 WFSライブラリは同一内容をブロードキャストする能力に優れているため、これによって転送効率の向上が見込めます。

指定した子機群のひとつでも揃わない限り親機からの応答が開始されなくなるため、 アプリケーションの設計によってはデッドロックが発生しうるという点に注意してください。

子機群から発行された読み込みリクエストの内容がどれかひとつでも異なる場合、同期設定はキャンセルされます。
