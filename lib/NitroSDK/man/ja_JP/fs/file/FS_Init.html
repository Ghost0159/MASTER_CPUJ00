FS_Init

構文

#include <nitro/fs.h>

void FS_Init( u32 default_dma_no );

引数

default_dma_no ファイルシステム内部のROM アーカイブが使用するDMAチャンネル
FS_DMA_NOT_USEを指定した場合、DMAは使用せずCPUによる転送を行います。
有効なDMAチャンネルを指定した場合、FS_SetDefaultDMA関数で設定を変更するまでこのDMAチャンネルを使用できません。
また、カードアクセスの転送元はIOレジスタなのでDMAチャンネル0を指定することはできません。

返り値

なし。

説明

ROM ファイルシステムを初期化します。
以下の処理を行います。

  • CARD_Init関数の呼び出しによるCARDライブラリ初期化
  • カレントディレクトリをROM アーカイブのルート ("rom:/") に設定
  • ROM アーカイブが使用するDMA番号の初期化

ROM アーカイブにDMAチャンネルを割り当てる場合の内部動作については、 CARD_ReadRomAsync関数の説明も参照ください。

注意

この関数で有効なDMAチャンネルを指定した場合、FS_SetDefaultDMA関数で設定を変更するまでこのDMAチャンネルを使用できません。
ファイルシステムはこのDMAチャンネルを使用して内部でNITRO ROM デバイスへのアクセスを行います。
CARD_ReadRom関数およびCARD_ReadRomAsync関数説明も参照ください。
