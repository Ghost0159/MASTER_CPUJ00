NITRO-SDK:
	NITRO-SDK 環境と AGBSDK 環境との共存方法について

					2004/02/24
					任天堂株式会社 環境制作グループ
------------------------------------------------------------------------------

　Cygwin では mount コマンドを利用することで、任意のディレクトリを単一の
ディレクトリツリー上にマッピングし仮想パスとして使用することができます。

  AGBSDK に付属の AGBGNUPro 環境においてこの仮想パスを利用している場合、
Cygwin インストールの際に AGBGNUPro 環境における仮想バスのルートディレクトリの
マウント位置が、新しくインストールされる Cygwin によって上書きされることにより
不具合が発生する可能性があります。

  AGB 開発環境の PATH 設定や Makefile 等の中で AGB ライブラリへのパスの記述が、
仮想パスを使用しない形式でなされていれば、新しい Cygwin で問題なく動作し、
NITRO-SDK の環境と共存可能であることが確認されています。

  もし新規インストールされた Cygwin において AGB 環境でのビルドに問題が起こり
ましたら、まず AgbGNUProBash.bat, AgbGNUProDos.bat, Makefile 等における環境変
数 AGBDIR や PATH の設定が AGB 環境へのディレクトリ名と一致しているかをご確認
ください。


(補足) NITROSDK_ROOT を .bashrc 等の環境設定ファイル内での設定

　NITROSDK_ROOT を Cygwin のインストール位置からの仮想パスで設定した場合、
コンパイラ等へのオプションとしてパスが正常に渡されない場合があります。

  このため、必ず

	"C:/(NitroSDKへのパス)"

  または

	"C:\(NitroSDKへのパス)"

  という様にドライブ名からの絶対パスで設定してください。
