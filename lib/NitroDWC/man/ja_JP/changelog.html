Note 8-16: HTTP通信/汎用ランキングのメモリ解放に関する不具合の修正

HTTP通信（GHTTP）ライブラリに起因する問題で、リクエスト発行時（DWC_GetGHTTPData関数、DWC_GetGHTTPDataEx関数、DWC_PostGHTTPData関数)に確保されるメモリが、完了コールバック発生時に解放されていたため、DWC_CancelGHTTPRequest関数を呼び出すと完了コールバックが発生せず、メモリが解放されない不具合がありました。
これは完了コールバックを待たずに、DWC_ShutdownGHTTP関数を呼び出しても発生します。
上記の不具合を、キャンセル時やライブラリの解放時に正しく行なわれるよう修正しました。

さらにDWC_GetGHTTPDataEx関数については、次のいずれかの条件で受信バッファが解放されない不具合があり、これも修正しました。

  • 引数bufferlenが0より大きい値、引数buffer_clearがTRUEという組み合わせで呼び出した場合
  • 通信エラーが発生した場合（引数bufferlenが0、引数buffer_clearがTRUEという組み合わせで呼び出した場合を除く）
※汎用ランキングライブラリは、内部でGHTTPライブラリを使用していますので、同じ不具合がありました。

Note 1-01: DWC_SetFriendStatusCallback関数の追加

友達の通信状態が変化した時に呼び出されるコールバック関数を設定する関数として、DWC_SetFriendStatusCallbackを追加しました。
上記コールバック関数は、通常は友達リスト同期処理のDWC_UpdateServersAsync関数の引数としてセットしますが、友達リスト同期処理が必要ない場合でも友達の通信状態をリアルタイムで把握したい場合に、本関数を用いてコールバック関数を設定してください。

Note 1-02: DWC_IsSendableReliable関数がFALSEを返す条件を修正

エラー発生中にDWC_IsSendableReliable関数を呼ぶとTRUEを返し、DWC_SendReliable*関数を呼ぶとFALSEを返すという不具合がありました。
このような場合に、DWC_IsSendableReliable関数がFALSEを返すように修正しました。

Note 1-03: 最大接続人数チェックの追加

DWCライブラリでは、マッチメイクによる最大接続人数は32なので、DWC_ConnectToAnybodyAsync, DWC_ConnectToFriendsAsync, DWC_SetupGameServerの各関数の接続人数の引数が、32を越えていないかどうかをチェックするASSERTを追加しました。

Note 1-04: 友達リスト最大長チェックの追加

DWCライブラリで扱える友達の最大人数は64なので、DWC_InitFriendsMatch, DWC_ConnectToFriendsAsyncの各関数の友達リスト長の引数が、64を越えていないかどうかをチェックするASSERTを追加しました。

Note 1-05: データ送受信用デバッグ設定関数のエラー処理を修正

DWC_SetSendDelay, DWC_SetRecvDelay, DWC_SetSendDrop, DWC_SetRecvDropの各関数内で、メモリ確保エラーが発生した場合にDWCエラーがセットされていませんでした。
これを正しく DWC_ECODE_TYPE_ALLOC の FATAL ERROR をセットするように修正しました。

Note 1-06: GHTTPライブラリ関数に失敗の返り値を追加

以下のGHTTPライブラリ関数に失敗の返り値を追加しました。

  • DWC_ProcessGHTTP：今までは必ずTRUEを返していましたが、エラー発生中はFALSEを返すように変更しました。
  • DWC_GHTTPPostAddString：今までは返り値がありませんでしたが、パラメータエラー、もしくはメモリ確保エラーの場合はFALSEを返すように変更しました。
  • DWC_PostGHTTPData：エラー発生中に本関数を呼び出した場合、今まではそのまま処理を続けていましたが、何もせずに新設したエラーDWC_GHTTP_IN_ERRORを返すように変更しました。
  • DWC_GetGHTTPData：エラー発生中に本関数を呼び出した場合、今まではそのまま処理を続けていましたが、何もせずに新設したエラーDWC_GHTTP_IN_ERRORを返すように変更しました。
  • DWC_GetGHTTPDataEx：エラー発生中に本関数を呼び出した場合、今まではそのまま処理を続けていましたが、何もせずに新設したエラーDWC_GHTTP_IN_ERRORを返すように変更しました。

Note 1-09: 親機DS通信切断による子機DSはまり不具合修正

マッチメイク中は接続中のDS間で一時的な親子関係が構築されます。
ここで親機となったDSが電源を切断するなどして通信できなくなった場合、接続済みであった子機DSがはまり状態となってしまう（マッチメイクのキャンセルはできる）という不具合がありました。
このような状態になった子機DSは、一定時間親機DSと通信できなければマッチメイクをやり直すように修正しました。

Note 1-10: DWC_CheckFriendKey関数の不具合修正

DWC_CheckFriendKey関数では、CRCで友達登録鍵の正当性をチェックしていましたが、友達登録鍵に含まれるGSプロファイルIDが負の値で、かつCRCも正しい友達登録鍵が入力された場合、本関数はTRUEを返すのに、友達リストの同期処理時にパラメータエラーが発生してしまうという不具合がありました。
これを、GSプロファイルIDが負の値の友達登録鍵に対しては、FALSEを返すように修正しました。

Note 1-11: データストレージサーバから自分のデータをロードする関数を追加

データストレージサーバから自分のデータをロードするDWC_LoadOwnDataAsync関数は、Privateデータしかロードできませんでした。
そのため、Publicデータをロードする DWC_LoadOwnPublicDataAsync関数、Privateデータをロードする DWC_LoadOwnPrivateDataAsync関数を追加しました。

Note 1-12: 友達指定ピアマッチメイクで友達の友達との接続を完全には禁止できない不具合を修正

DWC_ConnectToFriendsAsync関数の引数distantFriendにFALSEを指定しても、引数friendIdxListで指定した以外の友達と接続してしまうという不具合がありました。
これを、distantFriendにFALSEを指定した場合は、必ずfriendIdxListで指定した友達（friendIdxList = NULL なら友達全員）としか接続しないように修正しました。

Note 1-13: Fatal Error 71002 が発生する不具合を修正

通信状況が不安定になるなどでDWCのエラーが発生した後、内部処理を続けてしまい、エラーコード71002のFatal Errorが発生するという不具合がありました。
